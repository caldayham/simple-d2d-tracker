    2. Install dependencies: `npm install @supabase/supabase-js @supabase/ssr date-fns lucide-react sonner`

    3. Create `.env.local.example` with placeholder vars:
       - NEXT_PUBLIC_SUPABASE_URL=your-project-url
       - NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key

    4. Create `src/lib/types.ts` with TypeScript types matching the database schema:
       - `Session` type: id (string), user_id (string), started_at (string), ended_at (string | null), notes (string | null)
       - `Visit` type: id (string), session_id (string), latitude (number), longitude (number), address (string | null), audio_path (string | null), audio_mime_type (string | null), audio_duration_seconds (number | null), transcript (string | null), notes (string | null), recorded_at (string), created_at (string)

    5. Create `src/lib/audio.ts` with two exported functions:
       - `getBestAudioMimeType()`: Tries `audio/webm;codecs=opus`, `audio/mp4;codecs=aac`, `audio/mp4`, `audio/webm` in order using `MediaRecorder.isTypeSupported()`, returns first match or empty string.
       - `getFileExtension(mimeType: string)`: Maps mime to extension (mp4->m4a, webm->webm, ogg->ogg, default->bin).

    6. Create Supabase clients following the exact `@supabase/ssr` patterns from the research doc:
       - `src/lib/supabase/client.ts`: `createBrowserClient` for client components
       - `src/lib/supabase/server.ts`: `createServerClient` using cookies for server components/actions
       - `src/lib/supabase/middleware.ts`: `updateSession` helper that refreshes auth tokens

    7. Create `src/middleware.ts` following Pattern 5 from research:
       - Import `updateSession` from lib/supabase/middleware
       - Call `supabase.auth.getUser()` (NOT getSession -- security requirement)
       - Redirect unauthenticated users to `/login` (except `/login` itself and static assets)
       - Matcher excludes `_next/static`, `_next/image`, `favicon.ico`, `sw.js`, `icon-*.png`, `manifest.json`

    8. Create `src/app/login/page.tsx` as a client component:
       - Email + password form with large touch targets (min 48px)
       - Submit calls `supabase.auth.signInWithPassword()`
       - On success, redirect to `/record` via `router.push('/record')` and `router.refresh()`
       - Show error message on failure via state (no toast library needed here)
       - Simple, clean UI -- this is a single-user app, no signup form needed

    9. Create `src/app/page.tsx`: Simple redirect to `/record` (middleware handles auth redirect to login if needed).

    10. Create `src/app/record/layout.tsx`: Mobile-optimized layout with viewport meta for standalone PWA display. Import and render InstallPrompt component.

    11. Create `src/app/record/page.tsx`: Placeholder page with text "Recording UI coming soon" and the current session status. This will be built out in Plan 01-03.

    12. Create `src/app/manifest.ts` using Next.js built-in MetadataRoute.Manifest:
        - name: "Canvassing Companion", short_name: "Canvass"
        - start_url: "/record", display: "standalone"
        - theme_color: "#dc2626" (red for recording theme)
        - Icons: reference /icon-192x192.png and /icon-512x512.png

    13. Create `public/sw.js`: Minimal service worker with install (skipWaiting) and activate (clients.claim) handlers. No offline caching.

    14. Register the service worker in `src/app/layout.tsx` using a script tag or useEffect in a client component. Also add Toaster from sonner to the root layout.

    15. Create placeholder PWA icons: Generate simple 192x192 and 512x512 PNG files in /public (can be solid red squares with "CC" text -- functional placeholders).

    16. Create `src/components/shared/InstallPrompt.tsx` following the research example: Detects iOS + non-standalone mode, shows "Add to Home Screen" prompt banner at bottom of screen.

    17. Create `src/actions/sessions.ts` with server actions:
        - `createSession()`: Creates a new session row for the authenticated user, returns the session
        - `endSession(sessionId: string)`: Sets `ended_at` to now() on the session
        Both must call `supabase.auth.getUser()` for auth check.

    18. Create `supabase/migrations/001_schema.sql` with the exact SQL from the research doc:
        - sessions table (id UUID PK, user_id references auth.users, started_at, ended_at, notes)
        - visits table (id UUID PK, session_id references sessions ON DELETE CASCADE, latitude, longitude, address, audio_path, audio_mime_type, audio_duration_seconds, transcript, notes, recorded_at, created_at)
        - RLS enabled on both tables
        - Policy: users own their sessions (auth.uid() = user_id)
        - Policy: users own visits via session (session_id IN select from sessions where user_id = auth.uid())

    Note: The user will need to run this migration manually via Supabase Dashboard SQL editor or `supabase db push` after setting up their Supabase project. Include a comment at the top of the migration file explaining this.
