  <files>src/hooks/useAudioRecorder.ts</files>
  <action>
    Create a React hook `useAudioRecorder` that manages the full recording lifecycle with all iOS Safari pitfalls handled.

    **Hook API (return value):**
    ```typescript
    {
      isRecording: boolean,
      isPaused: boolean,
      duration: number,          // seconds elapsed since start
      mimeType: string,          // detected MIME type
      startRecording: () => Promise<void>,
      stopRecording: () => Promise<RecordingResult>,
      error: string | null,
    }

    type RecordingResult = {
      blob: Blob,
      mimeType: string,
      durationSeconds: number,
    }
    ```

    **Implementation details:**

    1. On `startRecording()`:
       - Call `navigator.mediaDevices.getUserMedia({ audio: { echoCancellation: true, noiseSuppression: true } })` to get the audio stream
       - Detect best MIME type using `getBestAudioMimeType()` from `src/lib/audio.ts`
       - Create `new MediaRecorder(stream, { mimeType })` if mimeType is non-empty, otherwise `new MediaRecorder(stream)`
       - Start with `recorder.start(10000)` -- 10-second timeslice for chunked recording (Pitfall 1 mitigation)
       - Acquire Wake Lock: `await navigator.wakeLock.request('screen')` in a try/catch (graceful fallback if unavailable)
       - Start a duration timer (setInterval every 1 second incrementing a ref)
       - Register `visibilitychange` listener: if `document.visibilityState === 'hidden'` and recording is active, call `stopRecording()` automatically (Pitfall 1 mitigation)

    2. `ondataavailable` handler: Push `e.data` to a chunks ref array if `e.data.size > 0`

    3. On `stopRecording()`:
       - Return a Promise that resolves when `recorder.onstop` fires
       - In onstop: assemble `new Blob(chunks, { type: mimeType })`, compute duration, release Wake Lock, stop all media stream tracks, clear the duration timer, remove visibilitychange listener
       - Return `RecordingResult`

    4. Cleanup on unmount: If recording is active, stop it. Release Wake Lock. Clear timers. Remove event listeners.

    **Critical details:**
    - Store chunks in a `useRef<Blob[]>` (not state -- no re-renders per chunk)
    - Store MediaRecorder, stream, wakeLock, timerId in refs
    - The visibilitychange handler must be added once on start and removed on stop (not in useEffect)
    - Handle the case where stopRecording is called while recorder.state is not 'recording' (it might be 'inactive' if visibilitychange already stopped it)
    - The duration counter should use a ref + state combo: ref for the interval, state for the display value

<task type="auto">
  <name>Task 2: Create useGeolocation hook</name>
  <files>src/hooks/useGeolocation.ts</files>
  <action>
    Create a React hook `useGeolocation` that manages GPS tracking with accuracy gating.

    **Hook API (return value):**
    ```typescript
    {
      position: { latitude: number; longitude: number } | null,
      accuracy: number | null,     // meters, null if no reading yet
      isWatching: boolean,
      isAccurate: boolean,         // accuracy < 20m
      error: string | null,
      startWatching: () => void,
      stopWatching: () => void,
    }
    ```

    **Implementation details:**

    1. `startWatching()`:
       - Call `navigator.geolocation.watchPosition(successCb, errorCb, options)`
       - Options: `{ enableHighAccuracy: true, timeout: 15000, maximumAge: 0 }`
       - Store the watchId in a ref

    2. Success callback:
       - Always update accuracy state with `pos.coords.accuracy`
       - Only update position state when `pos.coords.accuracy < 20` (accuracy gate -- Pitfall 4 mitigation)
       - When accuracy gate is met, store `{ latitude: pos.coords.latitude, longitude: pos.coords.longitude }`

    3. Error callback:
       - Map GeolocationPositionError codes to human-readable messages:
         - PERMISSION_DENIED: "Location permission denied"
         - POSITION_UNAVAILABLE: "Location unavailable"
         - TIMEOUT: "Location request timed out"

    4. `stopWatching()`: Clear the watch via `navigator.geolocation.clearWatch(watchId)`

    5. Cleanup on unmount: Stop watching if active.

    **Critical details:**
    - `isAccurate` is derived: `accuracy !== null && accuracy < 20`
    - Store watchId in useRef (not state)
    - The hook should be passive until `startWatching()` is called (no auto-start)
