  <files>src/actions/storage.ts, src/actions/visits.ts, src/lib/geocoding.ts</files>
  <action>
    **1. Create `src/lib/geocoding.ts`:**

    Export a `reverseGeocode(lat: number, lng: number)` function that:
    - Calls `https://nominatim.openstreetmap.org/reverse?format=jsonv2&lat=${lat}&lon=${lng}`
    - Sets `User-Agent: CanvassingCompanion/1.0 (contact@example.com)` header (Pitfall 6)
    - Returns `{ display_name: string, house_number?: string, road?: string, city?: string }` or null on error
    - Wraps in try/catch -- never throws, returns null on failure
    - Log errors to console but don't propagate (non-blocking geocoding principle)

    **2. Create `src/actions/storage.ts`:**

    Server action file with `'use server'` directive.

    Export `createSignedUploadUrl(filePath: string)`:
    - Create server Supabase client
    - Verify user is authenticated via `getUser()` -- throw if not
    - Call `supabase.storage.from('audio').createSignedUploadUrl(filePath)`
    - Return `{ signedUrl, path, token }` from the response
    - Throw on error with descriptive message

    **3. Create `src/actions/visits.ts`:**

    Server action file with `'use server'` directive.

    Export `createVisit(data: { session_id: string, latitude: number, longitude: number, audio_path: string, audio_mime_type: string, audio_duration_seconds: number })`:
    - Create server Supabase client
    - Verify user is authenticated via `getUser()`
    - Insert into `visits` table with all provided fields, `recorded_at` defaults to now
    - Return the created visit row
    - Throw on error

    Export `resolveAndUpdateAddress(visitId: string, lat: number, lng: number)`:
    - Call `reverseGeocode(lat, lng)` from lib/geocoding.ts
    - If result is not null, update the visit's `address` field with `result.display_name`
    - Return the resolved address string or null
    - This is designed to be called AFTER createVisit -- non-blocking (Pitfall: never block UI on geocoding)

    Note: `createSession` and `endSession` already exist in `src/actions/sessions.ts` from Plan 01-01. Do NOT recreate them.
