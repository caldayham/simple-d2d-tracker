<task type="auto">
  <name>Task 2: Wire recording page with full flow</name>
  <files>src/app/record/page.tsx</files>
  <action>
    Replace the placeholder record page with the full recording interface. This is a client component ('use client') that orchestrates everything.

    **Page structure (top to bottom):**
    1. SessionControls at the top
    2. GpsStatus bar below session controls
    3. RecordButton large and centered (the hero element)
    4. AddressDisplay below the record button
    5. UploadStatus at the bottom
    6. List of recent visits in current session (simple list: address or "Resolving...", duration, timestamp)

    **State management:**
    - `activeSession`: Session | null -- from createSession/endSession server actions
    - `isSessionLoading`: boolean
    - `lastAddress`: string | null -- from resolveAndUpdateAddress
    - `isResolving`: boolean
    - `pendingUploads`: number
    - `lastUploadStatus`: 'idle' | 'uploading' | 'success' | 'error'
    - `sessionVisits`: Array of { address, duration, recordedAt } for the current session display

    **Flow: Start Session**
    1. User taps "Start Canvassing"
    2. Call `createSession()` server action
    3. Store returned session in state
    4. Start GPS watching via `useGeolocation().startWatching()`
    5. Enable the record button

    **Flow: Start Recording**
    1. User taps record button (requires active session + GPS accurate)
    2. Call `useAudioRecorder().startRecording()`
    3. Button shows pulsing state + duration counter

    **Flow: Stop Recording**
    1. User taps record button again
    2. Call `stopRecording()` which returns `{ blob, mimeType, durationSeconds }`
    3. Capture current GPS position from useGeolocation
    4. Set upload status to 'uploading'
    5. Generate file path: `recordings/${activeSession.id}/${Date.now()}.${getFileExtension(mimeType)}`
    6. Call `createSignedUploadUrl(filePath)` server action
    7. Upload blob directly to Supabase Storage using the signed URL:
       ```typescript
       const supabase = createClient() // browser client
       await supabase.storage.from('audio').uploadToSignedUrl(path, token, blob, { contentType: mimeType })
       ```
    8. Call `createVisit({ session_id, latitude, longitude, audio_path: filePath, audio_mime_type: mimeType, audio_duration_seconds: durationSeconds })` server action
    9. Set upload status to 'success'
    10. Add visit to sessionVisits list
    11. Fire-and-forget: Call `resolveAndUpdateAddress(visit.id, latitude, longitude)` -- set isResolving=true, then update lastAddress and the visit in sessionVisits when it resolves
    12. On any error in steps 6-9: set upload status to 'error', show toast via sonner

    **Flow: End Session**
    1. User taps "End Session" (only when not recording)
    2. Call `endSession(sessionId)` server action
    3. Stop GPS watching
    4. Clear session state and visits list

    **Edge cases:**
    - Record button disabled when: no session, GPS not accurate yet, or upload in progress
    - If GPS becomes inaccurate mid-recording, continue recording (don't interrupt) -- use last known accurate position
    - Show toast (sonner) for: upload errors, GPS permission denied, microphone permission denied
    - Format duration display as MM:SS using simple math (no library needed)

    **Layout:**
    - Single column, full viewport height, padding for safe areas
    - Record button should be vertically centered or in the lower third for thumb reach
    - Use `pb-safe` or manual padding for iPhone home indicator area
  </action>
