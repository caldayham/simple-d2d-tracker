  <name>Task 1: Address-in-area server action using Overpass API</name>
  <files>src/actions/geocoding.ts</files>
  <action>
    Create `src/actions/geocoding.ts` as a server action file for geocoding operations related to route planning.

    **`findAddressesInArea` server action:**
    ```typescript
    'use server';

    export async function findAddressesInArea(
      polygon: Array<{ lat: number; lng: number }>
    ): Promise<Array<{ latitude: number; longitude: number; address: string }>>
    ```

    **Implementation using Overpass API** (OpenStreetMap's query API for geospatial data):

    1. Build an Overpass QL query to find buildings/houses within the polygon:
       ```
       [out:json][timeout:25];
       (
         way["building"]["addr:housenumber"](poly:"lat1 lng1 lat2 lng2 ...");
         node["addr:housenumber"](poly:"lat1 lng1 lat2 lng2 ...");
       );
       out center;
       ```
       The `poly` filter takes space-separated "lat lng" pairs. The `out center` returns the centroid of ways.

    2. Fetch from `https://overpass-api.de/api/interpreter` with POST body `data=<query>`.
       Set User-Agent header: `'CanvassingCompanion/1.0 (contact@cf.design)'`

    3. Parse results — each element has:
       - `tags["addr:housenumber"]` and `tags["addr:street"]` for the address
       - `center.lat`/`center.lon` (for ways) or `lat`/`lon` (for nodes) for coordinates

    4. Build address string: `"${housenumber} ${street}"` — skip elements without both housenumber and street.

    5. Deduplicate by address string (same house can appear as both node and way).

    6. Return sorted by address (natural sort so "2 Main St" comes before "10 Main St").

    **Error handling:** Return empty array on any failure (same non-throwing pattern as reverseGeocode). Log errors with console.error.

    **Rate limiting note:** Overpass API has no strict rate limit but requests should be reasonable. The polygon query is a single request per "Create Run Route" action, well within acceptable use.

    **Why Overpass over Nominatim:** Nominatim is designed for single-address lookups (forward/reverse geocoding). Overpass is designed for spatial queries ("find all X within this area"). This is the right tool for "all houses in a polygon."
