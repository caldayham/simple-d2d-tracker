    The component currently accepts `knocks: Array<{ latitude: number; longitude: number; address: string }>` for unsaved knocks. It needs to support two modes:

       - Each knock row gets `draggable`, `onDragStart`, `onDragOver`, `onDrop` handlers
       - `onDragStart`: Store the dragged index in a ref
       - `onDragOver`: `e.preventDefault()` to allow drop, add visual indicator (border highlight)
       - `onDrop`: Reorder the array by moving the dragged item to the drop position
       - Call `onReorder(newKnocks)` callback to update parent state

    2. **Saved mode** (new): Used when viewing a saved planned route's knocks. Accept `visits: Visit[]` and `onReorderKnocks: (orderedIds: string[]) => void` props.
       - Same drag-and-drop behavior
       - On drop, call `onReorderKnocks` with the new order of visit IDs

    Updated props interface:
    ```typescript
    interface PlannedKnockListProps {
      // Unsaved mode (Plan tab)
      knocks?: Array<{ latitude: number; longitude: number; address: string }>;
      onClear?: () => void;
      onReorder?: (knocks: Array<{ latitude: number; longitude: number; address: string }>) => void;
      // Saved mode (viewing planned route)
      visits?: Visit[];
      onReorderKnocks?: (orderedIds: string[]) => void;
    }
    ```
